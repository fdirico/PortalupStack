# GStack Document Generate Method Adaptation

Source: GStack `document-generate/SKILL.md`, MIT license.

## Diataxis Quadrants

- Tutorial: learning-oriented walkthrough.
- How-to: task-oriented procedure.
- Reference: complete factual surface.
- Explanation: rationale and tradeoffs.

## Research Before Writing

Map structure, entry points, configuration, implementation files, tests, related modules, public surface, dependencies, edge cases, and design decisions before writing docs.

## Recommended Order

1. Reference documentation first.
2. Explanation docs for why/tradeoffs.
3. How-to guides for specific tasks.
4. Tutorials when onboarding is needed.

## PortalUP Additions

- For customer docs, avoid internal jargon and include operational validation.
- For support docs, include runbook, rollback, escalation, and client message.
