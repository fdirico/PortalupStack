# GStack Document Release Method Adaptation

Source: GStack `document-release/SKILL.md`, MIT license.

## Release Documentation Audit

1. Pre-flight and diff analysis.
2. Coverage map: shipped surface vs reference/how-to/tutorial/explanation coverage.
3. Per-file documentation audit.
4. Apply clear factual updates.
5. Ask before risky narrative rewrites.
6. Changelog polish without clobbering history.
7. Cross-doc consistency and discoverability.
8. TODO cleanup.
9. Version bump only with explicit confirmation.

## PortalUP Additions

- Include installation, environment variables, DB migrations, runtime workers, customer communication, and rollback.
- Do not expose internal secrets, private client details, credentials, or infrastructure identifiers.
