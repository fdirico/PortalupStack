# UI Modernization Standard

## Questions Before Designing

Ask only what changes the outcome:

- Who uses this screen and how often?
- What are the top three actions?
- What data must be scanned quickly?
- Is this desktop-first, mobile-first, or both?
- Are there brand colors, typography, or client constraints?
- Should the interface feel operational, executive, commercial, analytical, or immersive?
- Are charts, 3D, sliders, maps, timelines, or realtime states useful for the task?

## Modern App Patterns

| Surface | Preferred pattern |
| --- | --- |
| Admin tool | dense layout, left nav or top tabs, compact tables, filters, clear actions |
| Dashboard | KPI strip, charts with comparisons, alert states, drilldowns, responsive grid |
| Client portal | polished navigation, guided workflows, clear status, friendly empty states |
| Data-heavy screen | table density controls, sticky headers, saved filters, bulk actions |
| Configuration | grouped forms, validation, previews, change summary, save/discard states |
| Command center | live cards, timelines, maps/graphs when useful, priority highlighting |

## Visual Quality Bar

- Typography: readable, restrained, no oversized default headings inside dense tools.
- Spacing: compact but breathable; avoid huge empty gaps.
- Buttons: small/medium, clear hierarchy, icons where useful.
- Menus: optimized, grouped, responsive, no clutter.
- Color: purposeful status and emphasis, not one-note palettes.
- Components: polished states for hover, focus, disabled, loading, empty, error.
- Charts: modern, labeled, comparable, not decorative.
- 3D: useful for product, process, spatial, or immersive contexts; avoid gratuitous effects.

## Validation

Use evidence where possible:

- Desktop and mobile screenshots.
- Check no text overlap or clipped buttons.
- Check keyboard focus and contrast.
- Check loading, empty, error, and success states.
- Confirm primary workflows are no more complex than before.
