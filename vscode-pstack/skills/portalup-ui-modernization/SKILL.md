# PortalUP UI Modernization

## Use

Use this skill to design or improve modern application interfaces. Prioritize professional product quality, efficient use of space, responsive behavior, visual hierarchy, accessibility, and polished interaction patterns.

Do not blindly create UI from an under-specified request when choices affect usability, brand, layout, or responsive behavior. Ask targeted questions or state assumptions.

## Workflow

1. Capture context: app type, users, workflows, brand, platform, devices, data density, accessibility needs, and target style.
2. If the request is broad or visual quality is central, read `references/ui-modernization-standard.md`.
3. Challenge weak requests: menus, dashboards, tables, charts, forms, and navigation need structure and responsive rules.
4. Define IA and layout: navigation, hierarchy, density, states, empty/error/loading behavior, mobile behavior.
5. Specify visual system: typography, spacing, color roles, components, buttons, cards, tables, charts, sliders, icons, and motion.
6. Propose implementation changes tied to files/components when code exists.
7. Define validation: screenshots, responsive checks, accessibility checks, interaction tests, and user-flow smoke checks.

## Checklist

- Target users and workflows are explicit.
- Missing design choices are asked or stated as assumptions.
- Layout uses space efficiently.
- Menus and actions are optimized for repeated work.
- Typography, spacing, contrast, states, and responsiveness are covered.
- Visual polish does not reduce usability or accessibility.
- Validation includes screenshots or responsive checks when implementable.
