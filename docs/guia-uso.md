# Guia de uso

## Patron recomendado

1. Indicar la skill.
2. Dar contexto suficiente.
3. Adjuntar archivos, logs o fragmentos relevantes.
4. Pedir una salida concreta.

Ejemplo:

```text
Usa $portalup-soporte-incidente con este log de WebSphere. Necesito diagnostico, validaciones y mensaje para cliente.
```

## Buenas practicas

- Pide que se separen hechos, supuestos e hipotesis.
- Para incidentes, comparte hora, ambiente, version, impacto y logs.
- Para propuestas, comparte cliente, dolor, alcance, restricciones y fecha objetivo.
- Para SQL, comparte motor, version, tamano de base, sintomas y planes de ejecucion si existen.
