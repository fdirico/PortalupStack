# Handoff Policy

## Objetivo

Permitir que otro agente o engine continue el trabajo sin depender del historial completo de conversacion.

## Formato minimo

```text
Objetivo
- ...

Estado actual
- ...

Decisiones tomadas
- ...

Archivos relevantes
- ...

Cambios realizados
- ...

Validaciones
- ...

Riesgos abiertos
- ...

Siguiente paso recomendado
- ...

Skills sugeridos para continuar
- ...
```

## Reglas

- El handoff debe ser entendible sin acceso al chat anterior.
- No incluir secretos ni credenciales.
- No mezclar pendientes con decisiones ya cerradas.
- Al cambiar de engine, indicar engine origen y engine destino.
